Map style is used to define the look and feel of a map. Itâ€™s written in JSON format and specifies how to:

-   Style the symbols, lines, polygons, background, hill shading, heatmap points, raster tiles, circles, and extruded polygons.
-   Specify which of them should be drawn.
-   Set the order in which to draw them.

<Blockquote>

  The structure of the map style can be found in the [Map Style Specification](https://developer.tomtom.com/map-display-api/documentation/mapstyles/map-style-specification) documentation.

</Blockquote>

Different applications may require different styles, so map visualisation can be aligned with brand identity or with specific use cases.
For example, the map in a tourist app will show POIs like museums and restaurants, while a truck navigation app will include road shields.
The Map Display module comes with two pre-defined styles, but it also supports loading custom styles.

# Pre-defined styles

The Map Display module provides pre-defined styles that can be used in the application.
They are provided in the [`StyleInput`](/maps-sdk-js/javascript/maps/documentation/api-reference/modules/map#StyleInput) as [`PublishedStyle`](/maps-sdk-js/javascript/maps/documentation/api-reference/modules/map#PublishedStyle).
A [`PublishedStyle`](/maps-sdk-js/javascript/maps/documentation/api-reference/modules/map#PublishedStyle) represents a style which is officially hosted by TomTom and easily accessible by its name.

These are the available [`PublishedStyle`](/maps-sdk-js/javascript/maps/documentation/api-reference/modules/map#PublishedStyle):

-   `standardLight` and `standardDark` - Styles made for browsing the map in light and dark mode respectively. The `standardLight` style is the default style used by the Map Display module.
-   `drivingLight` and `drivingDark` - Styles to use during navigation in light and dark mode respectively. These styles render routes in gray and show traffic flow and road shields.
-   `satellite` - This style uses satellite imagery to draw the map.
-   `monoLight`

# Loading a style

The style can be loaded when the map is initialized by specifying the `style` property of
the [`GOSDKMapParams`](/maps-sdk-js/javascript/maps/documentation/api-reference/modules/map#GOSDKMapParams) when initializing
an instance of [`GOSDKMap`](/maps-sdk-js/javascript/maps/documentation/api-reference/classes/map.GOSDKMap).

<Code>

```typescript
import { GOSDKMap, GOSDKMapParams } from "@anw/go-sdk-js/map";

const goSDKMapParams: GOSDKMapParams = {
    style: "standardDark"
}

const goSDKMap = new GOSDKMap(
    {
        // Container element defined in the corresponding HTML file
        container: document.getElementById("map") as HTMLElement,
        zoom: 14,
        minZoom: 2,
        center: [-0.12621, 51.50394]
    },
    goSDKMapParams
);
  ```

# Customizing a style

You can also create your own style to suit your application.

<Blockquote hasIcon>

  The easiest way to do this is to use the [Map Styler](https://developer.tomtom.com/maps-api/map-styler). It provides a user interface with real-time previews so you can see the effects of your changes.

</Blockquote>

Custom styles can be provided to the application in the [`StyleInput`](/maps-sdk-js/javascript/maps/documentation/api-reference/modules/map#StyleInput)
as a [`CustomStyle`](/maps-sdk-js/javascript/maps/documentation/api-reference/modules/map#CustomStyle).

{<a style={{display: 'block', margin: '0', padding: '0'}} id="_adding_hill_shading"></a>}

# Adding and filtering Points of Interest

The Map Display module contains a Points of Interest (POIs) layer, which is enabled by default. This layer can be excluded from the
map style with the `exclude` property of the [`GOSDKMapParams`](/maps-sdk-js/javascript/maps/documentation/api-reference/modules/map#GOSDKMapParams).

> For more information about the `exclude` property, see the [Map Configuration](/maps-sdk-js/javascript/maps/documentation/guides/map-display/map-configuration) guide.

The POI layer can be controlled during runtime with the [`VectorTilePOI`](/maps-sdk-js/javascript/maps/documentation/api-reference/classes/map.VectorTilePOIs) module.

```javascript
import { VectorTilePOIs } from "@anw/go-sdk-js/map";

const pois = await VectorTilePOIs.init(goSDKMap);
pois.setVisible(false);
```

It's also possible to filter shown POIs by category or classification group using the `filterCategories()` method on [`VectorTilePOI`](/maps-sdk-js/javascript/maps/documentation/api-reference/classes/map.VectorTilePOIs).
See the [Map POI filters](http://hosted-examples.maps-sdk-js.tomtom.com/main/map-poi-filters) example for this functionality.

# Adding hill shading

The Map Display module allows you to add style layers such as hill shading to the map style.

<img src="images/ss_hillshade.png" alt="Map with hill shading" />

<br/>

Hill shading is enabled by default. It can be excluded from the map style with the `exclude` property of the [`GOSDKMapParams`](/maps-sdk-js/javascript/maps/documentation/api-reference/modules/map#GOSDKMapParams).

> For more information about the `exclude` property, see the [Map Configuration](/maps-sdk-js/javascript/maps/documentation/guides/map-display/map-configuration) guide.

<Code>

```typescript
import { GOSDKMap, GOSDKMapParams } from "@anw/go-sdk-js/map";

const goSDKMapParams: GOSDKMapParams = {
    exclude: ["hillshade"]
    // other params...
};

const goSDKMap = new GOSDKMap(
    {
        // Container element defined in the corresponding HTML file
        container: document.getElementById("map") as HTMLElement,
        zoom: 14,
        minZoom: 2,
        center: [-0.12621, 51.50394]
    },
    goSDKMapParams
);
```

```javascript
import { GOSDKMap, GOSDKMapParams } from "@anw/go-sdk-js/map";

    // other params...
```

</Code>

To show or hide hill shading on runtime, use the [`VectorTilesHillshade`](/maps-sdk-js/javascript/maps/documentation/api-reference/classes/map.VectorTilesHillshade) class:

``` javascript
const hillshade = await VectorTilesHillshade.init(goSDKMap);
hillshade.setVisible(false);
```
