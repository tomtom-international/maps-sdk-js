The Map bundle is a [MapLibre GL JS - based library](/javascript/maps/documentation/guides/map/about-maplibre) and allows you to:
* visualize and configure a TomTom map in your web application
* easily display the results coming from TomTom services on the map (places, routes, geometries)
* easily interact with the key elements on the map
* display and interact with your own data

# Set up

1. Configure the project as described in [Project setup guide](/javascript/maps/documentation/overview/project-setup).
2. Make sure that after following the guide above you have both the SDK and MapLibre installed as dependencies.
3. Create an HTML page with a container element for the map (with an id):
 ```html
 <!DOCTYPE html>
 <html lang="en">
     <body>
         <div id="map" style="position: absolute; left: 0; top: 0; width: 100%; height: 100%"></div>
     </body>
 </html>
    ```
4. (If you need HTML elements on the map) Import MapLibre's CSS in your page. For instance, add this to your HTML head section:
 ```html
 <link href="https://unpkg.com/maplibre-gl@2.4.0/dist/maplibre-gl.css" rel="stylesheet" />
    ```
5. [Initialize a TomTom map](#initializing-a-tomtom-map)
6. [Discover and use some map modules](#map-modules)
<br/>

# Initializing a TomTom map

The [`TomTomMap`](/javascript/maps/documentation/api-reference/map-init.TomTomMap) object is the main entry point to add a *TomTom* map to your application.

Its main purpose is to initialize a TomTom map using MapLibre and serve as the base for further map modules to operate.

Importing the TomTomMap:
```javascript
import { TomTomMap } from "@anw/maps-sdk-js/map";
```

Minimal initialization (assuming the ID of your map HTML element is "map"):
```javascript
const map = new TomTomMap( { container: "map" } );
```

The TomTomMap is initialized via its constructor with:
* [MapLibreOptions](#maplibreoptions) - a required parameter based on a subset of the [options](https://maplibre.org/maplibre-gl-js-docs/api/map/#map-parameters) for a MapLibre Map.
* [TomTomMapParams](#tomtommapparams) - optional TomTom-specific parameters.
<br/>

## MapLibreOptions

The subset of [options](https://maplibre.org/maplibre-gl-js-docs/api/map/#map-parameters) for a MapLibre map.

For the TomTom SDK, The only required MapLibre parameter is *container*, where you specify the container element for the map canvas:
```javascript
const map = new TomTomMap( { container: "map" } );
```

Other common MapLibre init parameters are regarding start coordinates or bounds:

Init map with *center* and *zoom*:
```javascript
const map = new TomTomMap( { container: "map", center: [-0.12621, 51.50394], zoom: 14, minZoom: 2 } );
```

Init map with *bounds*:
```javascript
const map = new TomTomMap( { container: "map", bounds: [-0.12621, 51.50394, -0.102, 51.7] } );
```

The MapLibre *style* parameter is *ignored*. Refer to the TomTomMapParams [style](#map-style) parameter instead.

For more maplibre parameters, refer to its [documentation](https://maplibre.org/maplibre-gl-js-docs/api/map/#map-parameters)
<br/>

## TomTomMapParams

The TomTom Map parameters extend the global configuration and are focused on configuring your TomTom map.

Whereas the MapLibre parameters are more generic, these parameters are focused on the TomTom side of things,
saving you some time and plumbing compared to using MapLibre directly.

### Map style

The map style defines where the map data comes from and how it should look.
The TomTom SDK offers convenient ways to load TomTom map styles via its style parameter.

Minimal style input:
```javascript
const map = new TomTomMap({ container: "map" }, { style: "monoLight" });
```

Style input with exclusions:
```javascript
const map = new TomTomMap({ container: "map" }, { style: { id: "monoLight", exclude: ["trafficFlow"]}});
```

To learn more about map styles, refer to the [Map Styles Guide](/javascript/maps/documentation/guides/map/map-styles)
<br/>


# Map modules

In addition to the map itself, the Map bundle contains several individual modules that allow you to control and interact with key map parts:

## Map style - based modules
*These are modules that control part of the sources and layers coming via the loaded map style.*
* [Traffic Module](/javascript/maps/documentation/guides/map/traffic) - controls live traffic on the map.
* [POIs Module](/javascript/maps/documentation/guides/map/pois) - controls POIs on the map.
* [Hillshade Module](/javascript/maps/documentation/guides/map/hillshade) - controls hillshading on the map.

## Added data modules
*These are modules controlling data coming from services or custom sources.*
* [Places Module](/javascript/maps/documentation/guides/map/places) - controls added places on the map.
* [Geometry Module](/javascript/maps/documentation/guides/map/geometries) - controls added geometries on the map.
* [Routing Module](/javascript/maps/documentation/guides/map/routing) - controls added routes on the map.
