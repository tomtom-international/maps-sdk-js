# TomTom Maps SDK for JavaScript - Quickstart Guide

The TomTom Maps SDK for JavaScript is a comprehensive mapping library built on [MapLibre GL JS](/javascript/maps/documentation/guides/map/about-maplibre) that provides seamless integration with TomTom's mapping services and data. This SDK combines the power of TomTom's global mapping data with the performance and flexibility of modern web mapping technologies.

## What You Can Build

The Maps SDK enables you to create powerful mapping applications with advanced capabilities:

- **Interactive TomTom maps** with customizable styles and themes optimized for different use cases
- **Service integration** for places, routing, geocoding, and real-time traffic data
- **Advanced visualizations** for routes, geometries, and points of interest with rich styling options
- **User interactions** with built-in event handling and user interface components
- **Custom data overlays** that seamlessly integrate with TomTom's global map data

## Prerequisites and Setup

Before getting started, ensure you have the following requirements:

1. **Node.js and npm/yarn** installed for package management (Node.js 16+ recommended)
2. **TomTom API key** from the [TomTom Developer Portal](https://developer.tomtom.com/)
3. **Modern browser** with WebGL support for optimal map rendering performance

### Project Setup

Follow the complete [Project setup guide](/javascript/maps/documentation/overview/project-setup) for detailed installation instructions, or use this quick setup for immediate development:

```bash
npm install @cet/maps-sdk-js maplibre-gl
```

### Required Dependencies

The SDK architecture requires both the TomTom package and MapLibre as peer dependencies to ensure optimal performance and compatibility:

```json
{
  "dependencies": {
    "@cet/maps-sdk-js": "latest",
    "maplibre-gl": "^5.7.1"
  }
}
```

## Basic HTML Setup

Create a foundational HTML page structure with the required MapLibre CSS and a designated map container:

```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>TomTom Map Application</title>
    <link href="https://unpkg.com/maplibre-gl@5.7.1/dist/maplibre-gl.css" rel="stylesheet" />
    <style>
        #map { width: 100%; height: 400px; }
    </style>
</head>
<body>
    <div id="map"></div>
    <script type="module" src="./main.js"></script>
</body>
</html>
```

## Creating Your First Map

### Minimal Map Implementation

The most basic map implementation requires only API key configuration and container specification:

```javascript
import { TomTomConfig } from '@cet/maps-sdk-js/core';
import { TomTomMap } from '@cet/maps-sdk-js/map';

TomTomConfig.instance.put({
    apiKey: 'YOUR_API_KEY_HERE'
});

const map = new TomTomMap({
    container: 'map'
});
```

### Map with Initial Position

For applications requiring specific geographic focus, configure initial position and zoom level:

```javascript
const map = new TomTomMap({
    container: 'map',
    center: [4.9041, 52.3676], // Amsterdam coordinates [longitude, latitude]
    zoom: 12
});
```

### Map with Different Style

Customize the visual appearance using TomTom's built-in map styles:

```javascript
const map = new TomTomMap(
    { container: 'map' },
    { style: 'monoDark' }
);
```

## Map Styles and Customization

TomTom provides several pre-configured map styles optimized for different use cases and visual preferences:

```javascript
// Light theme (default) - optimal for general use
{ style: 'monoLight' }

// Dark theme - ideal for low-light environments
{ style: 'monoDark' }

// Satellite imagery - perfect for geographic analysis
{ style: 'hybrid' }
```

### Advanced Style Configuration with Additional Layers

Enhance your maps with real-time data layers for comprehensive location intelligence:

```javascript
const map = new TomTomMap(
    { container: 'map' },
    {
        style: {
            type: 'published',
            include: ['trafficIncidents', 'trafficFlow']
        }
    }
);
```

**Available Layer Modules:**
- **trafficIncidents**: Real-time traffic incident data
- **trafficFlow**: Live traffic flow visualization
- **pois**: Points of interest display
- **hillshade**: Terrain elevation shading

## Common Map Configuration Options

The TomTom Maps SDK provides extensive configuration options for fine-tuning map behavior and appearance:

```javascript
const map = new TomTomMap({
    container: 'map',
    center: [longitude, latitude],        // Initial map center
    zoom: 12,                            // Initial zoom level (2-18)
    minZoom: 2,                          // Minimum allowed zoom
    maxZoom: 18,                         // Maximum allowed zoom
    bounds: [west, south, east, north]   // Alternative to center/zoom
});
```

**Configuration Best Practices:**
- Use `center` and `zoom` for fixed initial views
- Use `bounds` for dynamic content that needs to fit specific geographic areas
- Set appropriate `minZoom` and `maxZoom` based on your application's requirements
