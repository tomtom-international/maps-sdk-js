# POIs (Points of Interest) Module Overview

The POIs Module in the TomTom Maps SDK provides comprehensive control over the display and filtering of Points of Interest on maps. This module enables dynamic management of POI visibility, categorization, and styling, helping users discover relevant locations and services in their area of interest.

## Understanding Points of Interest (POIs)

Points of Interest represent real-world locations and businesses that provide services or are of general interest to users. The TomTom POI database includes millions of locations worldwide, categorized and continuously updated to ensure accuracy and relevance.

### POI Categories and Classification

POIs are organized into logical categories that help users find specific types of locations:

**Transportation Infrastructure:**
- **Gas stations and charging stations** for vehicle refueling needs
- **Parking facilities** including garages, lots, and street parking
- **Public transit stations** for buses, trains, and subway systems
- **Airports and ports** for long-distance travel

**Food and Dining:**
- **Restaurants** spanning all cuisine types and price ranges
- **Cafes and coffee shops** for casual dining and beverages
- **Fast food establishments** for quick meal options
- **Bars and nightlife venues** for entertainment and social activities

**Accommodation Services:**
- **Hotels and motels** for temporary lodging needs
- **Hostels and budget accommodations** for cost-conscious travelers
- **Resorts and luxury properties** for premium experiences

**Essential Services:**
- **Healthcare facilities** including hospitals, clinics, and pharmacies
- **Financial institutions** such as banks, ATMs, and currency exchange
- **Government buildings** and public service offices
- **Emergency services** including police, fire, and medical facilities

**Shopping and Retail:**
- **Shopping centers and malls** for comprehensive retail experiences
- **Supermarkets and grocery stores** for daily necessities
- **Specialty shops** for specific products and services

## Module Architecture and Capabilities

The POIs Module serves as the primary interface for POI management with these core capabilities:

- **Dynamic visibility control** for showing and hiding POI layers based on user preferences
- **Category-based filtering** to display only relevant POI types for specific use cases
- **Custom styling options** for adapting POI appearance to application themes
- **Integration with other modules** for cohesive mapping experiences

### POI Data Sources and Quality

POIs in TomTom maps are sourced from comprehensive global databases that include:
- **Commercial business listings** with verified location and contact information
- **Public services and facilities** maintained by government and municipal sources
- **Transportation infrastructure** including real-time status updates
- **Tourist attractions and landmarks** with cultural and historical significance

## Basic POI Implementation

### Simple POI Display

Enable POI visualization with basic configuration:

```javascript
import { TomTomMap, POIsModule } from '@cet/maps-sdk-js/map';

// Create map with POIs included
const map = new TomTomMap(
    { container: 'map', center: [4.9041, 52.3676], zoom: 14 },
    {
        style: {
            type: 'published',
            include: ['pois']
        }
    }
);

// Get POI module instance
const pois = await POIsModule.get(map);
```

### Interactive POI Controls

Implement user controls for POI visibility management:

```javascript
// Toggle POI visibility
function togglePOIs() {
    pois.setVisible(!pois.isVisible());
    updatePOIControlsUI();
}

// Update UI to reflect current POI state
function updatePOIControlsUI() {
    const isVisible = pois.isVisible();
    document.getElementById('poi-toggle').classList.toggle('active', isVisible);
    document.getElementById('poi-status').textContent =
        isVisible ? 'POIs: Visible' : 'POIs: Hidden';
}
```

### Dynamic Category Management

Implement runtime category filtering for flexible POI display:

```javascript
class POICategoryManager {
    constructor(map) {
        this.map = map;
        this.activeCategories = new Set();
        this.poisModule = null;
    }

    async initialize() {
        this.poisModule = await POIsModule.get(this.map);
    }

    toggleCategory(category) {
        if (this.activeCategories.has(category)) {
            this.activeCategories.delete(category);
        } else {
            this.activeCategories.add(category);
        }
        this.updatePOIDisplay();
    }

    updatePOIDisplay() {
        // Implement category-specific visibility logic
        const shouldShowPOIs = this.activeCategories.size > 0;
        this.poisModule.setVisible(shouldShowPOIs);
    }
}
```

## POI Integration Patterns

### Search Integration with POI Display

Combine POI visualization with search functionality for comprehensive location discovery:

```javascript
import { search } from '@cet/maps-sdk-js/services';
import { PlacesModule } from '@cet/maps-sdk-js/map';

const placesModule = await PlacesModule.get(map);

// Search for specific POI types and display results
async function searchAndDisplayPOIs(query, category) {
    const results = await search({
        query,
        position: map.getCenter().toArray(),
        categorySet: [category],
        limit: 20
    });

    // Display search results as custom markers
    placesModule.addPlaces({
        id: `search-${category}`,
        places: results.results
    });

    // Ensure POI layer is visible for context
    pois.setVisible(true);
}
```

### Context-Aware POI Display

Implement intelligent POI visibility based on map zoom level and user context:

```javascript
class ContextualPOIManager {
    constructor(map, poisModule) {
        this.map = map;
        this.poisModule = poisModule;
        this.setupZoomBasedVisibility();
    }

    setupZoomBasedVisibility() {
        this.map.getMaplibreMap().on('zoom', () => {
            const zoom = this.map.getMaplibreMap().getZoom();

            // Show POIs at appropriate zoom levels
            if (zoom >= 13) {
                this.poisModule.setVisible(true);
            } else {
                this.poisModule.setVisible(false);
            }
        });
    }

    setLocationContext(userLocation, radius = 5000) {
        // Adjust POI visibility based on user location and area of interest
        const relevantCategories = this.getRelevantCategoriesForArea(userLocation);
        this.updatePOICategories(relevantCategories);
    }
}
```

## Performance Optimization

### Efficient POI Rendering

For applications with demanding performance requirements, implement optimized POI management:

```javascript
class OptimizedPOIManager {
    constructor(map) {
        this.map = map;
        this.poisModule = null;
        this.visibilityThrottle = null;
    }

    async initialize() {
        this.poisModule = await POIsModule.get(map);
        this.setupPerformanceOptimizations();
    }

    setupPerformanceOptimizations() {
        // Throttle visibility changes to prevent excessive redraws
        const throttledVisibilityUpdate = this.throttle(() => {
            this.updatePOIVisibility();
        }, 250);

        this.map.getMaplibreMap().on('move', throttledVisibilityUpdate);
        this.map.getMaplibreMap().on('zoom', throttledVisibilityUpdate);
    }

    throttle(func, wait) {
        let timeout;
        return function executedFunction(...args) {
            const later = () => {
                clearTimeout(timeout);
                func(...args);
            };
            clearTimeout(timeout);
            timeout = setTimeout(later, wait);
        };
    }
}
```
