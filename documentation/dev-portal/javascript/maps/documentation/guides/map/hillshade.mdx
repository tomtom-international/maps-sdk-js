# Hillshade Module Overview

The Hillshade Module in the TomTom Maps SDK provides terrain elevation visualization through semi-transparent shading that enhances the three-dimensional perception of topographic features. This module renders elevation data as a subtle overlay that reveals hills, valleys, mountains, and other terrain variations, making maps more visually engaging and geographically informative.

<img src="images/ss_hillshade.png" alt="Map with hillshading" />

## Understanding Hillshade Visualization

Hillshade visualization simulates the shadows cast by terrain features when illuminated by a light source, typically positioned from the northwest. This technique creates an intuitive perception of elevation and slope that helps users understand the physical geography of an area.

**Key Benefits:**
- **Enhanced terrain perception** making elevation changes visually apparent
- **Improved geographic context** for outdoor activities, urban planning, and navigation
- **Aesthetic enhancement** adding visual depth and professional cartographic quality
- **Subtle overlay design** that complements other map features without overwhelming them

**Technical Implementation:**
- **Vector tile-based rendering** ensuring sharp display at all zoom levels
- **Semi-transparent overlay** that works seamlessly with all map styles
- **Performance optimized** for smooth interaction and fast rendering
- **Global coverage** providing consistent hillshade data worldwide

# Initializing the Hillshade Module

## Basic Setup

First set up your project and map following the [Map quickstart guide](/javascript/maps/documentation/guides/map/quickstart) and ensure you have a running [TomTomMap](/javascript/maps/documentation/api-reference/map-init.TomTomMap) instance.

### Automatic Hillshade Integration

The quickest way to add hillshade to your map is through the style configuration during map initialization:

```javascript
import { TomTomMap } from "@cet/maps-sdk-js/map";

const map = new TomTomMap(
    { container: "map" },
    { style: { type: "published", include: ["hillshade"] } }
);
```

This approach automatically includes hillshade layers in the map style, making them available immediately when the map loads.

### Advanced Hillshade Control

For applications requiring dynamic hillshade management or runtime control, initialize the HillshadeModule separately:

```javascript
import { HillshadeModule } from "@cet/maps-sdk-js/map";

const hillshade = await HillshadeModule.get(map);
```

### Dynamic Hillshade Addition

Add hillshade to an existing map that was initialized without terrain visualization:

```javascript
import { HillshadeModule, TomTomMap } from "@cet/maps-sdk-js/map";

const map = new TomTomMap({ container: "map" }); // Map without hillshade

// Add hillshade later with style integration
const hillshade = await HillshadeModule.get(map, { ensureAddedToStyle: true });
```

The `ensureAddedToStyle` option dynamically adds hillshade layers to the existing map style, enabling terrain visualization without requiring map reinitialization.

## Configuration and Control

### Visibility Management

Control hillshade visibility dynamically based on application requirements or user preferences:

```javascript
import { HillshadeModule } from "@cet/maps-sdk-js/map";

// Initialize with custom visibility
const hillshade = await HillshadeModule.get(map, { visible: false });

// Toggle visibility programmatically
hillshade.setVisible(true);

// Check current visibility state
if (hillshade.isVisible()) {
    console.log('Hillshade is currently visible');
}
```

### Runtime Configuration Updates

Apply configuration changes dynamically without module reinitialization:

```javascript
// Hide hillshade for specific map modes
hillshade.applyConfig({ visible: false });

// Show hillshade for terrain-focused displays
hillshade.applyConfig({ visible: true });
```

### Combined Configuration During Initialization

Set up multiple configuration options during module initialization for complete control:

```javascript
const hillshade = await HillshadeModule.get(map, {
    visible: false,
    ensureAddedToStyle: true
});
```

## Integration Patterns

### Context-Sensitive Hillshade Display

Implement intelligent hillshade visibility based on map context and user interaction:

```javascript
// Show hillshade for outdoor/terrain contexts
function enableTerrainMode() {
    hillshade.setVisible(true);
    console.log('Terrain mode enabled with hillshade');
}

// Hide hillshade for urban navigation contexts
function enableNavigationMode() {
    hillshade.setVisible(false);
    console.log('Navigation mode - hillshade hidden for clarity');
}
```

### Zoom-Based Visibility Control

Automatically manage hillshade visibility based on map zoom level for optimal user experience:

```javascript
map.getMaplibreMap().on('zoomend', () => {
    const zoom = map.getMaplibreMap().getZoom();

    // Show hillshade at regional/country scales
    if (zoom < 8) {
        hillshade.setVisible(true);
    }
    // Hide hillshade at street/building scales
    else if (zoom > 14) {
        hillshade.setVisible(false);
    }
});
```

### Multi-Module Coordination

Coordinate hillshade with other map modules for cohesive visual experiences:

```javascript
import { HillshadeModule, POIsModule, TrafficIncidentsModule } from '@cet/maps-sdk-js/map';

const hillshade = await HillshadeModule.get(map);
const pois = await POIsModule.get(map);
const traffic = await TrafficIncidentsModule.get(map);

// Outdoor recreation mode
function enableOutdoorMode() {
    hillshade.setVisible(true);
    pois.setVisible(true);
    traffic.setVisible(false);
}

// Urban navigation mode
function enableUrbanMode() {
    hillshade.setVisible(false);
    pois.setVisible(false);
    traffic.setVisible(true);
}
```

## Event Handling

The Hillshade Module provides event handling capabilities for interactive terrain applications:

```javascript
const hillshade = await HillshadeModule.get(map);

// Handle clicks on terrain features
hillshade.events.on('click', (feature, lngLat) => {
    console.log('Terrain clicked at:', lngLat);

    if (feature.properties.elevation) {
        showElevationInfo({
            elevation: feature.properties.elevation,
            location: lngLat
        });
    }
});

// Handle hover for elevation preview
hillshade.events.on('hover', (feature, lngLat) => {
    if (feature.properties.slope) {
        showTerrainPreview(feature.properties, lngLat);
    }
});
```

## Performance Considerations

### Efficient Hillshade Management

For applications with varying hillshade requirements, implement efficient management patterns:

```javascript
// Lazy hillshade initialization
let hillshadeModule = null;

async function enableHillshade() {
    if (!hillshadeModule) {
        hillshadeModule = await HillshadeModule.get(map, {
            ensureAddedToStyle: true
        });
    }
    hillshadeModule.setVisible(true);
}

function disableHillshade() {
    if (hillshadeModule) {
        hillshadeModule.setVisible(false);
    }
}
```

### Memory and Rendering Optimization

Manage hillshade visibility strategically to optimize performance:

```javascript
// Hide hillshade when not needed to reduce rendering load
function optimizeForPerformance() {
    const currentZoom = map.getMaplibreMap().getZoom();

    // Only show hillshade at appropriate scales
    if (currentZoom >= 6 && currentZoom <= 12) {
        hillshade.setVisible(true);
    } else {
        hillshade.setVisible(false);
    }
}
```
