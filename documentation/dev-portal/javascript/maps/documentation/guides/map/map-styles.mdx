A map style is used to define the look and feel of a map. Itâ€™s written in JSON format and specifies how to:
*   Load the map data, specifying data sources and their URLs.
*   Define how the data should look like by defining layers for the different data sources:
    *   Style the symbols, lines, polygons, background, hill shading, heatmap points, raster tiles, circles, and extruded polygons.
    *   Specify which of them should be drawn.
    *   Set the order in which to draw them.

<Blockquote>
    The structure of the map style can be found in the [Map Style Specification](https://developer.tomtom.com/map-display-api/documentation/mapstyles/map-style-specification) documentation.
</Blockquote>

Different applications may require different styles, so map visualisation can be aligned with brand identity or with specific use cases.
For example, the map in a tourist app will show POIs like museums and restaurants, while a truck navigation app will include road shields.
The TomTom Map can load multiple pre-defined styles, but it also supports loading custom styles.

{<a style={{display: 'block', margin: '0', padding: '0'}} id="_pre_defined_styles"></a>}

# Pre-defined styles

The Map bundle provides pre-defined styles that can be used in the application.
They are provided in the [`StyleInput`](/javascript/maps/documentation/api-reference/map-init#StyleInput) as [`PublishedStyle`](/javascript/maps/documentation/api-reference/map-init#PublishedStyle).
A [`PublishedStyle`](/javascript/maps/documentation/api-reference/map-init#PublishedStyle) represents a style which is officially hosted by TomTom and easily accessible by its name.

The available [`PublishedStyle`](/javascript/maps/documentation/api-reference/map-init#PublishedStyle) are:

-   `standardLight` and `standardDark` - Styles made for browsing the map in light and dark mode respectively. The `standardLight` style is the default style.
-   `drivingLight` and `drivingDark` - Styles to use during navigation in light and dark mode respectively. These styles render routes in gray and show traffic flow and road shields.
-   `satellite` - This style uses satellite imagery to draw the map.


These styles are showcased in the [Map Config Playground](https://hosted-examples.maps-sdk-js.tomtom.com/main/map-config-playground/) example.

{<a style={{display: 'block', margin: '0', padding: '0'}} id="_loading_a_style"></a>}


# Loading a style

The style can be loaded when the map is initialized by specifying the `style` property of
the [`TomTomMapParams`](/javascript/maps/documentation/api-reference/map-init#TomTomMapParams) when initializing
an instance of [`TomTomMap`](/javascript/maps/documentation/api-reference/map-init.TomTomMap).




```javascript
import { TomTomMap } from "@anw/maps-sdk-js/map";
const map = new TomTomMap(
    { container: "map", zoom: 14, center: [-0.12621, 51.50394] },
    { style: "standardDark" }
);
```

Additionally, you can change the style of the map at runtime using the [`setStyle`](/javascript/maps/documentation/api-reference/map-init.TomTomMap#setStyle) method on the TomTomMap class.

```javascript
map.setStyle("drivingDark");
```

This is useful when you want to change the style at runtime. MapLibre will help with transitioning from the current to the new style smoothly.

{<a style={{display: 'block', margin: '0', padding: '0'}} id="_customizing_a_style"></a>}


# Customizing a style with Map Styler

You can also create your own style to suit your application.

<Blockquote hasIcon>
    The easiest way to do this is to use the [Map Styler](https://developer.tomtom.com/maps-api/map-styler). It provides a user interface with real-time previews so you can see the effects of your changes.
</Blockquote>

Custom styles can be provided to the application in the [`StyleInput`](/javascript/maps/documentation/api-reference/map-init#StyleInput)
as a [`CustomStyle`](/javascript/maps/documentation/api-reference/map-init#CustomStyle).

{<a style={{display: 'block', margin: '0', padding: '0'}} id="_excluding_modules_from_a_style"></a>}


# Excluding modules from a style

When setting the map style, you can exclude modules from the loaded map style with the `exclude` property in [`TomTomMapParams`](/javascript/maps/documentation/api-reference/map-init#TomTomMapParams).
For example, excluding [hill shading](/javascript/maps/documentation/guides/map/map-styles#_adding_hill_shading) and [traffic flow](/javascript/maps/documentation/guides/map/traffic) from the map style:

```javascript
import { TomTomMap } from "@anw/maps-sdk-js/map";

const map = new TomTomMap(
    {
        // Container element defined in the corresponding HTML file
        container: "map",
        zoom: 14,
        minZoom: 2,
        center: [-0.12621, 51.50394]
    },
    {
        style: {
          type: "published",
          exclude: ["hillshade", "traffic_flow"]
        }
    }
);
```

# Adding and filtering Points of Interest

The styles from the Map bundle contain a Points of Interest (POIs) layer. This layer is enabled by default, but can be excluded from the
map style with the `exclude` property of the [`TomTomMapParams`](/javascript/maps/documentation/api-reference/map-init#TomTomMapParams).

<img src="images/ss_poi_layer.png" alt="Map with POI layer enabled" />

<br/>

The POI layer visibility can be controlled during runtime with the [`POIsModule`](/javascript/maps/documentation/api-reference/map-POIs.POIsModule) module.

```javascript
import { POIsModule } from "@anw/maps-sdk-js/map";

const pois = await POIsModule.init(map);
pois.setVisible(false);
```

It's also possible to filter shown POIs by category or classification group using the `filterCategories()` method on [`POIsModule`](/javascript/maps/documentation/api-reference/map-POIs.POIsModule).
See the [Map POI filters](http://hosted-examples.maps-sdk-js.tomtom.com/main/map-poi-filters) example for this functionality.

# Adding hill shading

The Map bundle allows you to add style layers such as hill shading to the map style.

<img src="images/ss_hillshade.png" alt="Map with hill shading" />

<br/>

Hill shading is enabled by default. It can be excluded from the map style with the `exclude` property of the [`TomTomMapParams`](/javascript/maps/documentation/api-reference/map-init#TomTomMapParams).

To show or hide hill shading on runtime, use the [`HillshadeModule`](/javascript/maps/documentation/api-reference/map-hillshade.HillshadeModule) class:

``` javascript
const hillshade = await HillshadeModule.init(map);
hillshade.setVisible(false);
```
